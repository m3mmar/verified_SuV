VeriFast Help > Error messages

Error message

No matching heap chunks: chunk pattern

Context

When consuming a field assertion or predicate assertion.

Cause

No chunk matching the field assertion or predicate assertion was found in the symbolic heap.

Notes

  • The program element that causes the error is shown in red, with a double underline.
  • The chunk pattern shown in the error message is an evaluated version of the assertion being consumed.
  • Consumption of a field assertion or predicate assertion occurs implicitly in the following circumstances:
    • a field access consumes (and then produces) the field predicate;
    • a free statement consumes the field predicates and the malloc block predicate of the struct instance being freed;
    • an open statement consumes the predicate being opened.

Suggested solutions

Ensure that a matching chunk is present in the symbolic heap. Suggested approaches:

  • Force the current function's caller to provide the chunk. To do so, mention the chunk in the current function's requires clause.
  • If the chunk is a predicate chunk, close the predicate.
  • If the chunk is contained within a chunk that is present in the symbolic heap, open the containing chunk.
  • If the chunk can be derived from existing chunks, call a lemma function that derives the chunk.
